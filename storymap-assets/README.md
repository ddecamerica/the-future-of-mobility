# StoryMap Assets

This folder contains production-ready assets for **The Future of Mobility** GIS StoryMap, a comprehensive exploration of connected mobility vision for Palm Beach County.

Assets are organized for use through GitHub Pages, allowing images and reference documents to be linked directly from ArcGIS StoryMaps or other web-based presentation tools.

## About These Assets

These 42 production assets tell the data story behind Palm Beach County's connected mobility opportunity, with focus on the Palm Beach International Airport to Downtown West Palm Beach Connector. They support a narrative around regional growth, transportation patterns, tourism economy, and mobility infrastructure needs.

### The Data Stories

**Population & Growth Trends**
- New resident migration (90,000 new residents)
- Population projection for 2030 (1.63 million)
- Multiple design variations for flexible StoryMap presentation

**Commute & Travel Patterns**
- Drive-alone commute rates (77% of workers)
- Highlights opportunity for alternative mobility
- Multiple design variations to show data emphasis

**Tourism & Economic Activity**
- Annual visitors (106 million in FY 2024-25)
- Palm Beach International Airport passengers (85 million annually)
- Demonstrates regional economic significance

**Visual Branding**
- DDEC project identity (primary and reverse versions)
- Consistent professional presentation
- Integration across all StoryMap sections

## Naming Rules

Use simple, stable, web-safe file names. Once a URL is used in the StoryMap, the file name cannot change without breaking the link.

### Rules

1. Use lowercase letters only
2. Use hyphens instead of spaces
3. Avoid underscores, parentheses, accents, and special characters
4. Keep names descriptive enough to understand file purpose
5. Use numbering when multiple versions or variations exist
6. Use `-mini` suffix only for intentionally smaller or compact versions
7. Do not rename or move assets after their URLs are used in the StoryMap

### Good Examples

```
future-mobility-cover-hero-01.png
pbc-stat-drive-alone-77pct-01.png
pbc-stat-drive-alone-77pct-01-mini.png
ddec-logo-primary-black.png
future-mobility-storymap-guidance.pdf
```

### Avoid

```
Future Mobility Cover Final.png
PBC Stat FINAL FINAL (2).png
DDEC Logo White Version New.png
The-Future-of-Mobility-Story-Map-Guidance.pdf
```

## Updating Assets

### For Minor Corrections

If an asset URL has been added to the StoryMap and you need to make minor corrections (colors, text, small layout adjustments), replace the existing file using the same file name. The URL remains valid and no StoryMap updates are needed.

### For Major Revisions

For significant design changes or data updates, create a new versioned file and intentionally update the StoryMap link. This prevents accidentally breaking production references.

**Example:**
```
pbc-stat-drive-alone-77pct-01.png (current version in use)
pbc-stat-drive-alone-77pct-02.png (new major revision)
```

### URL Stability

Once an asset URL has been added to the StoryMap, keep the file name and folder location unchanged. This ensures production reliability and prevents broken links.

## Design System & Standards

### Visual Hierarchy

- **Noto Sans Bold**: Major section titles and hero headings
- **Noto Sans SemiBold**: Navigation labels, captions, and interface text
- **Noto Sans Medium**: Body copy, descriptions, and narrative text

### Asset Quality Standards

| Category | Recommended Format | Max Size |
|----------|-------------------|----------|
| Cover | JPG, PNG | 2MB |
| Sections | JPG, PNG | 1.5MB |
| Maps | JPG, PNG | 2MB |
| Logos | PNG (transparent) | 500KB |

### Image Resolution

- **Cover & Section Images**: Minimum 1920×1080px for web quality
- **Maps**: 2000×2000px minimum for clarity
- **Logos**: Vector format preferred; if raster, use PNG with transparency

## Dynamic GIS Maps

Dynamic, interactive maps are managed directly inside the GIS/StoryMap environment. Static map exports do not need to be stored in this repository unless they are required as standalone images, backup visuals, or non-interactive reference graphics.
